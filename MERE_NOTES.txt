SECTION 1 — KYA BANAYA HUM NE?
========================================================================

Indus Analytics ka purana ERP (Estimo Prime — Thomson client) VB.NET
WebForms + MSSQL pe bana hua tha. Bahut slow, modern UI nahi, multi-
tenancy nahi. Hum ne usse ground-up rebuild kiya — naya naam:

    Prathvi's ERP (URL-safe: prathvis-erp)

Yeh ek MULTI-TENANT SaaS hai matlab:
- Ek hi codebase
- Ek hi database (Postgres)
- Lekin alag-alag companies (tenants) apna-apna data alag-alag dekhte hain
- Har request mein tenant_id check hota hai

MVP scope (v0.1):
  ✓ Login / Auth
  ✓ Purchase Requisition (PR) — create, edit draft, submit, approve, reject
  ✓ Purchase Order (PO) — create, GST split, line items, financial summary
  ✓ Dashboard
  ✓ Items & Vendors master
  ✓ GRN + Gate Entry scaffolds

SECTION 2 — TECH STACK (kya kya naya tha mere liye)
========================================================================

FRONTEND (apps/web folder):
---------------------------
  Next.js 14 (App Router)
    - React framework — pages aur routing built-in
    - "App Router" = naya routing system (app/ folder mein files)
    - SSR (Server-Side Rendering) + Client Components dono support

  TypeScript
    - JavaScript + types
    - Compile time pe errors pakad leta hai
    - VS Code mein autocomplete bahut achha

  Tailwind CSS
    - Utility-first CSS — class="bg-blue-500 p-4" type
    - Custom design tokens packages/ui/tokens/circle.css mein
    - Theme switch single file change se possible

  React Hook Form + Zod
    - Form state management + validation
    - Zod schema FE aur BE dono mein same (packages/shared)


BACKEND (apps/api folder):
--------------------------
  Node.js 20 + Express 4
    - HTTP server framework
    - Middleware pattern: cors, helmet, auth, rbac

  TypeScript
    - Same as frontend

  tsx (TypeScript Execute)
    - TS code direct run karta hai bina compile kiye
    - dev mein "tsx watch" auto-reload deta hai
    - Production mein bhi tsx hi chalata hai (build step skip)

  Drizzle ORM
    - Type-safe SQL builder
    - SELECT * FROM users WHERE id = $1 ki jagah:
      db.select().from(users).where(eq(users.id, id))
    - TypeScript types schema se auto-generate
    - Migrations apne aap banti hain (drizzle-kit)

  Zod (validation)
    - Schema banao -> input validate hota hai automatically
    - Errors humare frontend mein dikhte hain inline

  argon2 (password hashing)
    - bcrypt se better, modern
    - Native build chahiye (pnpm allow karna padta hai)

  jsonwebtoken (JWT)
    - Access token: 15 minutes valid
    - Refresh token: 7 days valid
    - Stateless — server pe session store nahi

  pino (logging)
    - Fast JSON logger
    - pino-http req/res automatic log karta hai


DATABASE:
---------
  Neon (Serverless Postgres)
    - Manage karne ki zaroorat nahi
    - Free tier mein 500MB
    - WebSocket driver (@neondatabase/serverless) — port 443 use karta hai
      INSTEAD OF 5432, kyunki ISPs aksar 5432 block karte hain


MONOREPO TOOLS:
---------------
  pnpm workspaces
    - Ek repo mein multiple packages (apps/web, apps/api, packages/shared)
    - Shared dependencies dedup ho jati hain (disk save)

  Turborepo
    - "pnpm dev" se saare apps parallel mein chalte hain
    - Build caching: agar code nahi badla to dobara build nahi karega

  Folder layout:
    indus-erp-v2/
    |-- apps/
    |   |-- api/          (Express backend)
    |   |-- web/          (Next.js frontend)
    |-- packages/
    |   |-- shared/       (Zod schemas, types — FE+BE shared)
    |   |-- ui/           (design tokens, Tailwind preset)
    |-- README.md
    |-- DEPLOYMENT.md
    |-- pnpm-workspace.yaml
    |-- turbo.json

SECTION 4 — JO PROBLEMS AAYI AUR KAISE FIX KIYE
========================================================================

(Yeh sabse important section hai. Real-world deploy karne mein yeh sab
hota hai. Inhe yaad rakhna future mein.)

1. PowerShell execution policy ne pnpm block kar diya
   FIX: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned -Force

2. setup.ps1 mein emoji tha, PowerShell 5.1 ne parse error diya
   REASON: PS 5.1 UTF-8 without BOM ko ANSI samajhta hai
   FIX: Script ko ASCII-only rewrite kiya

3. drizzle-kit "Cannot find module './tenants.js'" deta tha
   REASON: drizzle-kit CommonJS require() use karta hai aur .ts files
           ko .js extension se resolve nahi kar paata
   FIX: Saare imports se ".js" extension hata diya (bulk find-replace)

4. Migrations local pe ETIMEDOUT — Neon port 5432 timeout
   REASON: Ghar ka ISP outbound port 5432 block karta hai
   FIX: @neondatabase/serverless + ws (WebSocket driver) install kiya
         Port 443 use hota hai (HTTPS jaisa) — kabhi block nahi hota

5. pnpm 11 ne ERR_PNPM_IGNORED_BUILDS diya argon2/esbuild ke liye
   REASON: pnpm 11 native builds explicitly approve karne ki bolta hai
   FIX: pnpm-workspace.yaml mein onlyBuiltDependencies list add ki

6. Git init galat folder mein hua (parent d:/25 May/ mein)
   PROBLEM: Legacy VB.NET folder bhi GitHub pe push ho jata
   FIX: Stray .git delete karke indus-erp-v2/ ke andar fresh init kiya

7. Render build fail — pnpm 11.3 + Node 20 = ERR_UNKNOWN_BUILTIN_MODULE
   REASON: pnpm 11.3 ek naya node:* module use karta hai jo Node 20 mein
           nahi hai (Node 22.5+ mein aaya hai)
   FIX: package.json mein "packageManager": "pnpm@9.15.0" set kiya
         Lockfile v9 hai jo pnpm 9 bhi support karta hai

8. Render start fail — tsx not found
   REASON: NODE_ENV=production ke saath pnpm devDependencies skip kar
           deta hai. tsx aur drizzle-kit dono devDeps mein the.
   FIX: apps/api/package.json mein tsx + drizzle-kit ko devDependencies
         se uthake dependencies mein move kiya

9. Render free tier mein Shell nahi mil raha
   PROBLEM: db:migrate aur db:seed kaise chalayein?
   FIX: Migrations local pe already chal chuki thi same Neon DB pe
         (production aur dev same DB), so re-run ki zaroorat nahi thi

10. Vercel apps/web se deploy par "npm install" use kar raha tha
    REASON: apps/web folder mein pnpm-lock.yaml nahi, woh root mein hai
    FIX: Vercel REST API call karke project ka rootDirectory="apps/web"
          aur sourceFilesOutsideRootDirectory=true set kiya

11. Vercel build "No Next.js version detected"
    REASON: Vercel root ki package.json dekh raha tha
    FIX: Same as #10 — rootDirectory set hone ke baad theek hua

12. Vercel "next build" mein TypeScript errors fail kar raha tha
    REASON: apps/web mein kuch pre-existing type errors hain jo runtime
            affect nahi karte
    FIX: next.config.mjs mein typescript.ignoreBuildErrors=true,
          eslint.ignoreDuringBuilds=true set kiya
          (Future mein in errors ko theek karna hai)

13. CORS — Vercel se API call pe error
    REASON: API ko nahi pata Vercel URL allowed hai
    FIX: Render Environment Variables mein WEB_ORIGIN add kiya:
          WEB_ORIGIN=https://prathvis-erp.vercel.app

SECTION 9 — JO TECH MAINE PEHLI BAAR USE KIYE (apni learning notes)
========================================================================

Mere liye bilkul nayi cheezein jo iss project mein seekhi:

1. MONOREPO ARCHITECTURE
   Pehle har project alag repo banata tha. pnpm workspaces se pata
   chala ki ek hi repo mein FE + BE + shared code rakhna kitna
   convenient hota hai. Type changes ek jagah, dono apps mein
   reflect.

2. ZOD SCHEMAS SHARED FE-BE
   Zod schema packages/shared mein likha. Wahi schema FE pe form
   validation karta hai aur BE pe API request validate karta hai.
   No duplication. Single source of truth.

3. DRIZZLE ORM (vs Sequelize/Prisma)
   Type-safe SQL builder. SQL ke close hai but TypeScript types
   automatic. Prisma generate step nahi.

4. JWT AUTH (vs session cookies)
   Stateless authentication. Server pe session store nahi.
   Access token: 15 min, Refresh token: 7 days.
   Frontend localStorage mein store karta hai.

5. MULTI-TENANCY (path-based)
   URL mein /t/<slug>/dashboard format. Har query mein
   tenant_id filter. RBAC permissions check.

6. SERVERLESS POSTGRES (Neon)
   Pehle managed Postgres = $$. Neon free tier serverless mein
   chalata hai. WebSocket driver port 443 — firewall friendly.

7. NEXT.JS APP ROUTER (vs Pages Router)
   app/ folder structure. Server Components by default.
   "use client" directive jab needed.

8. VERCEL CLI DEPLOY
   Dashboard click karne ki zaroorat nahi. Terminal se sab kuch.
   vercel link, vercel env add, vercel --prod.

9. CSS DESIGN TOKENS
   --primary, --bg, --text-default — CSS variables in tokens.
   Tailwind ke saath integrate. Theme swap = single import change.

10. GST SPLIT LOGIC
    Indian tax system: intrastate = CGST + SGST, interstate = IGST.
    isInterstate flag drive karta hai computation.

11. PAISE STORAGE (integer)
    Rs 123.45 ko 12345 paise integer mein store. Floating-point
    rounding errors avoid. Display time pe /100 karke dikhao.

12. PRE-EXISTING TYPECHECK ERRORS — IGNORE OR FIX?
    Production deploy block ho raha tha. Decision: ignore for now,
    fix in follow-up. Runtime affected nahi hota.
